Автоматизированные тесты для сайта **Stellar Burgers**.

---

## 📋 Что проверяется

- ✅ Главная страница — URL и все элементы (хедер, кнопки, заголовок, формы, корзина)
- ✅ Переход на страницу логина по кнопке «Войти в аккаунт»
- ✅ Вход с некорректными данными (`test`/`test`) — проверка ошибки
- ✅ Вход с корректными данными — перенаправление на главную
- ✅ Выход из профиля — перенаправление на страницу логина
- ✅ Проверка логов браузера на наличие ошибок (SEVERE) — 404, CORS, JS-ошибки

## 🚀 Установка и запуск

### 1. Клонирование

```bash
git clone https://github.com/ParamonovMO/stellarburgers-tests.git
cd stellarburgers-tests
```

### 2. Виртуальное окружение

```bash
python -m venv venv
source venv/bin/activate      # Linux/macOS
venv/Scripts/activate         # Windows
```

### 3. Установка зависимостей

```bash
pip install -r requirements.txt
```

### 4. Запуск тестов

```bash
pytest tests/ -v
```

---

## 📊 Allure-отчёты

### Локальный запуск

```bash
pytest tests/ --alluredir=allure-results
allure serve allure-results
```

### Автоматическая генерация в CI

При каждом пуше в `main` или `dev`, а также при создании Pull Request, GitHub Actions автоматически:
- запускает все тесты,
- генерирует Allure-отчёт,
- публикует его на GitHub Pages.

Актуальный отчёт всегда доступен по ссылке на бейдже в начале файла.

## 📌 Предположения и ограничения

### Предположения

- Сайт **Stellar Burgers** доступен по указанному URL (`https://stellarburgers.education-services.ru/`).
- Тестовый пользователь существует (`paramonov.maxim@vk.com` / `13542Qwe`).
- После выхода система перенаправляет на страницу логина.

### Ограничения

- Тесты запускаются только в Google Chrome (можно расширить параметризацией браузеров).
- Таймаут ожидания элементов — 5 секунд (можно увеличить при необходимости).
- Данные пользователя хранятся в коде (не загружаются из переменных окружения).
- Скриншоты при падении не реализованы (могут быть добавлены как улучшение).
- Можно расширить фикстуры (добавить авторизацию для уменьшения блока кода в `TC_PROFILE_001`)
